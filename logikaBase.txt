Тут прописана вся логика базового варианта:

Импорты

Делаем app
Назначаем ему SessionMiddleware

Создаем переменную templates и сохраняем в нее Jinja2Templates

Переменная с названием БД

Функция подключения к БД
    Создаем подключение
    Меняем способ возвращения данных на строки
    возвращаем соеденение

Функция создания нашей БД
    подключаемся к БД
        с помощью соеденения выполняем код в скобках
            Создаем 2 таблицы

Вызываем создание БД

get /:
    редиректим к /login
 
get logout:
    удаляем куки
    редиректим к /login

get register:
    возвращаем шаблон register.html - без ошибки
    
get login:
    возвращаем шаблон login.html - без ошибки    


post register (login,password,fio,email,phone):
    подключаемся к БД
        сохраняем все данные в юзер
    редиректим к /login


post login (login,password):
    Если логин == Админ
        Если пароль == КорокНЕТ
            сохраняем в куках что это Админ
            редиректим к /admin
        Если пароль не совпал:
            возвращаем шаблон login.html - с ошибкой что данные неверные    

    подключаемся к БД
        user = все данные из юзер где логин совпал  - fetchone()
    Если user вообще есть И при этом пароль из user == password (Который мы получили с фронта)
        сохраняем в куках id данного пользователя
        редиректим к /profile
    возвращаем шаблон login.html - с ошибкой что данные неверные    

get profile
    Берем id пользователя из куков
    Если его нет
        редиректим к /login    
    
    подключаемся к БД
        в request_list сохраняем все данные о курсах текущего пользователя - fetchall()
    возвращаем шаблон profile.html с данными из request_list

2 переменных - ВСЕ КУРСЫ  и ВСЕ СПОСОБЫ ОПЛАТЫ

get create_request
    Берем id пользователя из куков
    Если его нет
        редиректим к /login    

    возвращаем шаблон create_request.html +  ВСЕ КУРСЫ иВСЕ СПОСОБЫ ОПЛАТЫ 


post create_request (course,date,payment)
    Берем id пользователя из куков
    Если его нет
        редиректим к /login    

    подключаемся к БД
        сохраняем все данные в рекуэстс + id текущего пользователя
    
    редиректим к /profile

get admin
    Смотрим  из куков пользователь админ
    Если нет
        редиректим к /login    
    
    подключаемся к БД
        в request_list сохраняем все данные о курсах + логин и фио где из 2 таблиц где id пользователя = курсам к котором привязан пользователь - fetchall()
    возвращаем шаблон admin.html с данными из request_list
